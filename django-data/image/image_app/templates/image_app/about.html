About IMAGE-InjectTool

Introduction

One key challenge of that IMAGE addresses is the integration and transparent use of the vast information stored within more than 60 gene banks/genetic collections spanning 20 European countries, together with the collection of newly generated data. Information to be collected and integrated through the IMAGE portal is currently stored in local databases such as the ones at gene banks, research institution, breeding companies. A wide range of systems is used to store data representing genebank and genomic collections: from simple excel files to large national well-structured databases.

Submission of historical data to BioSamples, the reference archive for IMAGE, is facilitated to invite participation and to ensure standardization of the data. Cryoweb is a web-based software for the documentation and storage of gene bank data. Currently, the majority of National gene banks are using Cryoweb to manage their information. In France, the CRB-Anim database collects and store information for reproductive and genomic material of domestic animals (mammals, birds, fishes and shellfishes) in the country. Other countries have developed relational databases (i.e. using MySQL management system) to manage their collections.

The IMAGE-InjectTool project is a tool which will make easier uploading data into Biosample from the user point of view.

Methods

We developed a single Inject tool helping gene bank/genomic collections managers to enhance, standardise, tag and submit their gene bank data to a Common Data Pool that integrates all gene bank records from across Europe. The Inject tool uses a well-defined metadata ruleset ensuring high quality and comparable data across the diverse collections originating in different storage formats and languages.

The system relies on a docker-compose instance of some custom docker images: the user will interact on a web application rendered using NgINX, while on the backend a PostgreSQL instance stores user data and a django image render pages in Bootstrap 4 in order to display data and proving tools able to do data manipulation. A Celery and a Redis instances perform long tasks such as the data validation process, data submissions to biosample and monitoring of biosample submission statuses.

All those processed are addressed by the system, requiring user intervention only to fix data error and to provide mandatory data as described in IMAGE-metadata rules. User will be able to upload data from their cryoweb instances or excel spreadsheet defined in the metadata rules context. The project is manly written in Python3 using the test-driven approach system, which consists in defining test cases and coding in order to pass each test resulting in a more stable system. For this reason, data submitted through InjectTool to Biosamples will never persists and will be deleted from Biosamples test servers every week. We are doing our best to develop and testing stuff in order to achieve a persistent submission of data in Biosample servers
