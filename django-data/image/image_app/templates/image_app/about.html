About IMAGE-InjectTool

Introduction

One key challenge of that IMAGE addresses is the integration and transparent use of the vast information stored within more than 60 gene banks/genetic collections spanning 20 European countries, together with the collection of newly generated data. Information to be collected and integrated through the IMAGE portal is currently stored in each individual's local databases at gene banks, research institutions, breeding companies etc. which utilizes a wide range of systems : from simple excel files to large national well-structured databases.

Cryoweb is a web-based software for the documentation and storage of gene bank data. Currently, the majority of National gene banks are using Cryoweb to manage their information. In France, the CRB-Anim database collects and stores information for reproductive and genomic material of domestic animals (mammals, birds, fishes and shellfishes) in the country. Other countries have developed relational databases to manage their collections. Using InjectTool, data derived from different sources can be standardized and validated against IMAGE-metadata standards and finally submitted to EBI BioSamples, which has been chosen as the reference archive for all IMAGE data.

The IMAGE-InjectTool is a tool which makes it easier to archive data into BioSamples from the user point of view.

Methods

The InjectTool has been developed to help gene bank/genomic collections managers to enhance, standardise, tag and submit their gene bank data to a Common Data Pool that integrates all gene bank records from across Europe. The InjectTool uses a well-defined metadata ruleset ensuring high quality and comparable data across the diverse collections originating in different storage formats and languages.

The system relies on a docker-compose instance of some custom docker images: the user will interact on a web application rendered using Nginx, while on the backend a PostgreSQL instance stores user data and a Django image renders pages in HTML using Bootstrap 4 in order to display data and proving forms able to do data manipulation. A Celery and a Redis instances perform long tasks such as the data validation process, data submissions to biosample and monitoring of biosample submission statuses.

No user intervention is required to validate/submit data through InjectTool, however a manual revision is required if the validation process fails against IMAGE-metadata standards. User will be able to upload data from their Cryoweb dump files or excel spreadsheet defined in the metadata rules context. The project is mainly written in Python 3 using the test-driven approach. Submission statuses

Processes like data import, data validation and submission to Biosamples requires time and can't be accomplished in the same time required to get an HTML page. For this reason, InjectTool performs long tasks asynchronously in the background, in order to render web pages fast. However, you can't modify your data while performing such long tasks, otherwise you could invalidate valid-data during a submission process. In such way, we have defined different submission statuses relying on which tasks are accomplished on your data. Statuses may change when tasks are completed, you should try to reload the page in order to see updates in statuses. We plan to develop status updates in the visualized page without user intervention. Current statuses are:

  • Loaded: Data were correctly imported from your data sources. You need to validate them before submitting to biosample
  • Waiting: Some tasks are currently running on your data, you can't edit or delete your data or perform other tasks like validation or submission to biosample. All links or buttons related to such changes are disabled, and you have to wait in order to modify your data again. Try to reload the page in order to update submission status.
  • Error: This status should never be seen. This status could be determined by an issue in InjectTool. You may receive an email describing what is gone wrong, please forward this email to InjectTool developers by describing also what you are doing when you got this particular status.
  • Ready: Submission has been validated with success and could be submitted to Biosamples
  • Need revision: Submission has issues in validation or in Biosamples submission. You need to check error message and fix your data accordingly. Modifing or deleting your data in any other submission states, will result in a Need revision status. You have to validate your submission with success to get a Ready state and being able to submit your data to Biosample.
  • Submitted: Data have been submitted to Biosamples using API. The system is waiting for Biosamples response in order to get a Biosamples ID.
  • Completed: Data were submitted to biosample with success and have received a Biosamples ID. Congratulations! now your data are published in Biosamples archives.

InjectTool test site

This site is currently in testing phase. We currently submit data to BioSamples test server. Therefore data submitted through InjectTool will be deleted from BioSamples test servers every week. We are doing our best to shorten the test period while making the system as reliable as possible. We welcome any feedback/comment/suggestion which can be submitted by opening a new issue to make InjectTool better.
